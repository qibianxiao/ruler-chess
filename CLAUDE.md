# 尺棋 (Ruler Chess) — 项目概要

## 项目信息
- **路径**: `ruler-chess/`
- **技术**: 纯 HTML + CSS + JS，零依赖，静态网页
- **文件**: index.html (100行) + styles.css (530行) + game.js (620行)
- **公网**: https://46b636a05af5dc.lhr.life (localhost.run SSH隧道，服务器进程需运行)

## 游戏规则摘要
- 16×16 棋盘，起点左下角 (15,0)，终点右上角 (0,15)
- 两个棋子：白皇后（AI控制，player 0）和红皇后（人类玩家，player 1），轮流行动
- 刻度尺 0-20，大格 0/5/10/15/20，骰子 1-6
- 刻度尺默认向20移动，边界 (0/20) 反弹
- 只有停在大格时棋子才能移动，移动步数=骰子点数
- 每步走相邻格（上下左右），不能走出棋盘
- 人类点击高亮相邻格移动，AI自动选择曼哈顿最短路径

## 道具（全部隐藏）
- 小炸弹×8：向起点方向强制移动3格
- 大炸弹×2：直接返回起点
- 梯子×2：向终点方向强制移动3格
- 药水×4：刻度尺调到最近大格（平局选高处）
- 不生成在起点/终点，不重叠，不连锁触发

## AI 设计
- 策略 `chooseBestMove()`: 每步选离终点曼哈顿距离最近的相邻格，平局优先棋盘中央
- 同步执行，零延迟：所有步数瞬间算完，最后一次性渲染结果
- AI 回合期间跳过中间渲染和道具特效动画，极致响应速度
- AI回合期间roll按钮禁用，棋盘不可点击

## 道具特效
- 格子闪光（颜色对应道具类型）+ 棋盘震动（炸弹）+ 中央弹窗（1.6秒）

## 关键状态 (state)
- `currentPlayer`: 0=白皇后AI, 1=红皇后人类
- `phase`: 'roll' | 'move' | 'ai-thinking' | 'gameover'
- `aiPlayer: 0`, `aiActive: false`
- `items[]`: { type, row, col }

## 部署
- 本地: `python -m http.server 8080` (在 ruler-chess/ 目录)
- 公网隧道: `ssh -R 80:localhost:8080 nokey@localhost.run`
